Cisco Catalyst 9200L IOS-XE 17.9.8 Getestet März 2026

Factory Reset Anleitung

Vollständiger Guide — von gesperrtem Switch zu sauberem Factory Default

Vorbereitung — das brauchst du

  • Console-Kabel am Switch (RJ45-Port an der Rückseite)
  • PuTTY offen — COM-Port auswählen, Speed: 9600, Connection Type: Serial
  • USB-Stick (FAT32) mit cat9k_lite_iosxe.17.09.08.SPA.bin im Root-Verzeichnis
  • USB-Stick in den Switch gesteckt
1
In den ROMMON booten

Ziel: Switch startet und zeigt switch: in PuTTY.

  1. Switch neustarten — Strom weg, wieder rein
  2. Sofort den rechten grauen Knopf (>•< Symbol, rechts neben Cisco-Logo) gedrückt halten
  3. Knopf halten bis PuTTY folgendes zeigt:
switch:
Falls der Switch normal durchbootet: nochmal von vorne. Knopf früher drücken — direkt beim Einschalten.
2
Startup-Config ignorieren

Damit der Switch beim nächsten Boot die alte Config (und Passwörter) ignoriert.

set SWITCH_IGNORE_STARTUP_CFG=1
switch: (kein Fehler = gut)
3
Vom USB booten

Zuerst prüfen welcher USB-Port erkannt wird:

dir usbflash0:

Falls Fehler kommt:

dir usbflash1:

Du siehst die .bin Datei → booten (mit dem Port der funktioniert hat):

boot usbflash0:cat9k_lite_iosxe.17.09.08.SPA.bin
— oder —
boot usbflash1:cat9k_lite_iosxe.17.09.08.SPA.bin

Viele #### erscheinen → dauert ~3 Min. Enter drücken wenn Press RETURN to get started! erscheint.

Switch> (kein Login nötig)
4
Config löschen
enable
write erase

Bestätigung mit Enter.

Erase of nvram: complete
5
Factory Reset durchführen
USB-Stick vorher rausziehen! Dann formatiert der Switch nur seine interne Flash — der Stick bleibt unberührt.
factory-reset all

Er fragt zweimal nach Bestätigung → jeweils Enter drücken. Dauert ~3 Min.

switch: (Switch ist jetzt komplett clean)
6
Wieder vom USB booten

USB-Stick wieder rein stecken. Dann USB-Port prüfen (wie Schritt 3) und booten:

boot usbflash0:cat9k_lite_iosxe.17.09.08.SPA.bin
— oder —
boot usbflash1:cat9k_lite_iosxe.17.09.08.SPA.bin

Switch bootet (~3 Min). Enter drücken wenn Press RETURN to get started! erscheint.

Switch>
7
Setup-Wizard & Enable-Passwort vergeben

Der Switch fragt beim ersten Start:

Would you like to enter the initial configuration dialog? [yes/no]:

→ no eingeben und Enter drücken

Dann fragt er nach einem neuen Enable-Passwort (mind. 10 Zeichen, 1 Großbuchstabe, 1 Zahl) — vergib eines und bestätige es. Danach erscheint:

[2] Save this configuration to nvram and exit.

→ 2 eingeben

Switch>
8
Enable-Passwort direkt wieder entfernen

Da wir keinen Passwortschutz wollen, entfernen wir es direkt:

enable
conf t
no enable secret
end
write memory

Falls er nach einem Passwort fragt → Enter drücken (leer lassen). Ab jetzt geht enable ohne Passwort.

9
IOS auf Flash kopieren

Damit der Switch ohne USB booten kann:

enable
copy usbflash0:cat9k_lite_iosxe.17.09.08.SPA.bin flash:
— oder mit usbflash1 —
enable
copy usbflash1:cat9k_lite_iosxe.17.09.08.SPA.bin flash:
Falls Invalid input → usbflash1: statt usbflash0: verwenden (oder umgekehrt)

Bei der Dateinamenfrage einfach Enter drücken. Dauert ~3 Min.

474388619 bytes copied in XX secs
10
Boot von Flash konfigurieren & speichern
conf t
boot system flash:cat9k_lite_iosxe.17.09.08.SPA.bin
end
write memory
Building configuration... [OK]

USB-Stick rausziehen, dann neustarten:

reload

→ Enter bestätigen

11
Erfolgreich testen
enable
show version | include System image
System image file is "flash:cat9k_lite_iosxe.17.09.08.SPA.bin"

Fertig ✓

Steht da flash: (nicht usbflash) → Switch ist auf Factory Default, bootet eigenständig und hat keine Passwörter. Bereit für den nächsten Switch!

Troubleshooting

dir usbflash0: gibt Fehler
→ usbflash1: statt usbflash0: verwenden (und umgekehrt)
ERROR: Integrity Check Failed
→ Die .bin Datei auf dem USB ist beschädigt. Datei nochmal kopieren, sicher auswerfen, nochmal booten.
Switch landet immer im ROMMON
→ packages.conf fehlt/kaputt. Einfach vom USB booten und Schritt 8–9 durchführen.
media drive "usbflash0:" not present
→ USB-Stick nochmal raus und rein stecken, dann dir usbflash0: nochmal probieren.
copy ... flash: → Invalid input
→ Den anderen USB-Port verwenden: usbflash0 ↔ usbflash1 tauschen.
